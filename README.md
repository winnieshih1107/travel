## 工作報告

### 專案概述

本專案為個人旅行日誌網站，以互動式世界地圖為核心，整合旅遊日記、照片相簿與地理位置標記，呈現完整的旅行足跡記錄。所有資料儲存於瀏覽器 `localStorage`，無需帳號、無需伺服器，隨開隨用。

---

### 功能特色

| 功能 | 說明 |
|------|------|
| 🗺 互動式世界地圖 | Leaflet.js 驅動，CartoDB Voyager 英文地名圖磚 |
| 📍 地圖插旗 | 點擊地圖任意位置快速新增日誌，自動識別地名 |
| 🔍 地名自動定位 | 輸入地點名稱，Nominatim API 自動填入緯度經度 |
| 📅 日期區間 | 支援出發日期 + 返回日期，卡片顯示 `YYYY-MM-DD ~ YYYY-MM-DD` |
| 🗒 行程說明 | 獨立欄位記錄行程重點摘要，顯示於日誌卡片 |
| 📝 旅遊日記 | 自由書寫故事、感受、難忘瞬間 |
| 🖼 多張照片 | 上傳多張圖片，自動壓縮至最大 1200px，1/2/3/4+ 格排版 |
| 💡 燈箱瀏覽 | 點擊照片全螢幕瀏覽，左右切換 |
| ✎ 編輯 / 刪除 | 每篇日誌均可修改或移除 |
| 📊 統計數字 | 首頁即時顯示旅誌篇數、地點數、照片張數 |
| 📱 響應式設計 | 桌機與行動裝置皆適用 |

### 開發重點

- **資料模型設計**：每筆日誌含 `id`、`dateStart`、`dateEnd`、`location`、`desc`（行程說明）、`emoji`、`lat`、`lng`、`body`、`photos[]`
- **地圖整合**：點擊地圖 → 記錄座標 → 自動反向地理編碼填入地名 → 開啟新增表單
- **地名搜尋**：輸入地點名稱 650ms 去抖動後呼叫 Nominatim，自動填入座標並顯示確認訊息
- **照片壓縮**：透過 Canvas resize 減少 localStorage 佔用，支援批次上傳與單張移除
- **排版邏輯**：照片數量 1/2/3/4+ 分別套用不同 grid 排版，確保視覺一致
- **舊資料相容**：一次性 migrate 舊版 `wj_pins` / `wj_journals` 資料至新統一格式
- **日期區間顯示**：自動判斷單日 vs 區間，backward compatible 支援舊版單一 `date` 欄位

## 使用說明

### 新增日誌

點擊右上角「**✏️ 新增日誌**」，填寫以下欄位：

| 欄位 | 說明 |
|------|------|
| 出發日期 * | 行程開始日期（必填） |
| 返回日期 | 行程結束日期（選填，不填則顯示單日） |
| 地點名稱 | 輸入後自動帶入座標，在地圖插旗 |
| 行程說明 | 簡短摘要，例：東京 3 天 2 夜，走訪淺草、富士山 |
| 內文 | 自由書寫旅行故事與感受 |
| 照片 | 可上傳多張，自動壓縮儲存 |
| 地圖圖釘 Emoji | 自訂地圖上的圖示（預設 📍） |

---

### 地圖插旗

- **方法 A**：在地點名稱欄輸入地名 → 系統自動搜尋座標 → 儲存後地圖出現圖釘
- **方法 B**：直接點擊地圖任意位置 → 自動開啟新增表單並填入座標與地名

---

### 查看日誌

- **地圖**：點擊圖釘查看地點照片與日期
- **日誌列表**：捲動頁面至「旅行日誌」區塊，依出發日期由新到舊排列
- **照片燈箱**：點擊任意照片全螢幕瀏覽，可左右切換

---

### 編輯 / 刪除

每張日誌卡片底部有「**✎ 編輯**」和「**✕ 刪除**」按鈕，可隨時修改或移除。

---

### 資料說明

- 所有資料（含照片）儲存於瀏覽器 `localStorage`，**換裝置不會同步**
- 清除瀏覽器快取將**永久刪除**所有日誌，建議定期截圖備份
